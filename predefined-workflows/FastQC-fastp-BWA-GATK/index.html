Detailed Guide 详细说明

Workflow Overview 工作流概览

This tool triggers a comprehensive genomic analysis pipeline for cancer risk prediction. The workflow includes:

  • Quality Control: FastQC for initial quality assessment
  • Read Trimming: fastp for adapter trimming and quality filtering
  • Alignment: BWA-MEM for mapping reads to reference genome
  • Variant Calling: GATK best practices pipeline
  • Analysis: Variant annotation and risk prediction

Input Requirements 输入要求

FASTQ Files: Should be in the specified input directory. Both single-end and paired-end reads are supported.

File naming convention: sample_R1.fastq.gz, sample_R2.fastq.gz for paired-end

Reference Genome 参考基因组

The reference directory should contain:

  1. FASTA file (e.g., hg38.fa)
  2. Index files for BWA and GATK
  3. Known variant databases (dbsnp, 1000G, etc.)
  4. Target regions BED file (if applicable)

Frequently Asked Questions 常见问题

Authentication Issues 认证问题

Q: Why do I need a GitHub Token?
A: The workflow runs on GitHub Actions. A token with "repo" scope is required to trigger workflows and access private repositories.

Q: My token is not working, what should I do?
A: Ensure your token has the "repo" scope. Generate a new token if needed. Tokens expire after certain periods.

Common Errors 常见错误

Q: "Workflow not found" error
A: Check that the workflow file gatk.yaml exists in the repository and you have permission to access it.

Q: "Path does not exist" error
A: Verify that all input paths exist and are accessible from the workflow environment. Use absolute paths.

Performance Tips 性能提示

Thread number: Set appropriate thread count based on your server capacity. Higher threads = faster processing but more resource usage.

Job naming: Use descriptive names to easily identify runs in GitHub Actions dashboard.
