Detailed Guide

Pipeline Overview

This tool is a pipeline for processing and analyzing scRNAseq data. People can use the default parameters or choose parameters by themselves. The pipeline includes steps:

Data Loading---Quality Control---Normalization+Feature Selection---Diamensionality Reduction---Batch Correction---Clustering---Marker Identification---Annotation---Cell Compositional Analysis

Input Requirements

Directory Structure:
(1) Under the root input directory, each group should have its own folder. The names of folders should be consistent with the desired group names. For example, "covid" and "normal"
(2) Under each group's folder, each sample should have its own folder. The names of folders should be consistent with the desired sample ids. For example, "nCoV_1" and "Normal_1"
(3) Each sample folder contain the data files for that sample. Please see the specific structure example in the "Examples".

Files Format: This pipeline supports MEX format files and HDF5 format files. The recommended MEX format file should contain three compressed files, which must be named barcodes.tsv.gz, features.tsv.gz and matrix.mtx.gz. HDF5 format files need to end with .h5.

Folder Permission: The root input folder must be set to permissions 757 or 777 to allow read and write access for the Docker container.

Output Structure

Output Path: All files are in the output directory.

File Content: The output includes 11 png plots, 2 csv files, 1 html file and 1 rds file. The specific structure can be seen in the "Examples".

Parameter Tips

Use: The default parameters can be used directly for analysis. Please check all blanks have values and just click start button to run the pipeline.

Sample_QC: If want to set a separate quality control threshold for a sample, please click the "Add Sample" button to do so.

JOB_NAME: It can only contain a-z, A-Z, 0-9, _, ., -

GitHub PAT: Required for private repositories. Create token with "repo" scope at: GitHub Token Settings.

Frequently Asked Questions

Authentication Issues

Q: My Secret Key(Token) is not working, what should I do?
A: Check your Secret Key. It may have expired.

Common Problems

Q: "Workflow not found" error
A: Check that the workflow file run-scrna.yaml exists in the repository and you have permission to access it.

Q: "Path does not exist" error
A: Verify that all input paths exist and are accessible from the workflow environment. Use absolute paths. And please check if the input format complies with the requirements.

Q: "Please fill all required fields" error
A: This reminds that there are some blanks in the form that might not have been filled in. Especially, check if the Token has been filled in.

Q: "No space left" error
A: Please wait a second and try again.

Q: Why sample qc does not work?
A: Maybe the sample name is wrong. Please check the sample name.
